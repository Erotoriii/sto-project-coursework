# ADR-003: Вибір стеку Node.js + React + TypeScript

## Статус
Accepted

## Контекст
Для монолітної системи потрібен сучасний Backend (REST API, автентифікація касирів, робота з БД).
Система повинна витримувати до 1000 запитів на хвилину під час старту продажів на очікувані фільми.
Необхідно мати чіткий API-контракт (OpenAPI) для майбутньої розробки мобільного додатка для клієнтів.
Бюджет: 0 грн (open-source).
## Рішення
Використовувати мову програмування Python 3.12 + фреймворк FastAPI + ORM SQLAlchemy.

## Обґрунтування
1. Автоматична генерація: FastAPI генерує інтерактивну документацію OpenAPI (Swagger) автоматично на основі коду, що ідеально підходить для вимог нашого проєкту та синхронізації з Frontend.
2. Висока продуктивність: використання асинхронності (asyncio) дозволяє легко обробляти сотні паралельних запитів на перевірку вільних місць.
3. Велика екосистема: готові бібліотеки в PyPI для генерації PDF-чеків, відправки email-квитків та інтеграції з платіжними шлюзами.

## Альтернативи
| Варіант | Чому відхилено |
|---------|---------------|
| Python (Django/FastAPI) + Vue.js | Команда не має досвіду з Python; два різних синтаксиси (Python + JS) |
| Java (Spring Boot) + Angular | Надмірна складність для проєкту; повільна розробка через boilerplate код; високе споживання пам'яті |
| PHP (Laravel) + Blade | Застарілий підхід (server-side rendering); складніше реалізувати SPA |

## Наслідки
### Позитивні
- Повний збіг коду та API-документації в реальному часі (API-First підхід).
- Швидка реалізація бізнес-логіки завдяки лаконічності Python.

### Негативні
- Необхідність суворо слідкувати за тим, щоб не блокувати асинхронний Event Loop важкими обчисленнями (наприклад, під час генерації великого місячного звіту).
